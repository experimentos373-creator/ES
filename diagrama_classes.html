  %% Relacoes
  Team "1" *-- "0..26" Player : contains
  Match "1" *-- "0..*" Event : logs
  Match "1" *-- "1" MatchStats : includes
  Match "1" *-- "4" Ticket : offers
  Match "0..*" o-- "4" Referee : officiatedBy
  Match "0..*" --> "2" Team : competes
  Group "1" o-- "4" Team : groups
  Hotel "0..*" o-- "0..1" Team : accommodates
  User "0..*" --> "0..1" Team : manages

Notas Arquiteturais (Preparacao Java)

  • Encapsulamento: Todos os atributos estao definidos como privados (-), acessiveis via metodos publicos (+) ou getters/setters.
  • Composicao vs Agregacao: Relacoes fortes (como Jogo ↔ Eventos ou Jogo ↔ Bilhetes) usam composicao (*--). Relacoes fracas (como Jogo ↔ Arbitros ou Hotel ↔ Equipa) usam agregacao (o--).
  • Tipagem: As colecoes estao tipadas (ex: List~Player~) para refletir o uso de Generics em Java.
